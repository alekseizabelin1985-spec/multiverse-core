# 🏛️ CityGovernor

> **CityGovernor управляет городскими структурами и обеспечивает их функционирование.**

## 🎯 Назначение

- Управление городскими сущностями и структурами
- Обеспечение функционирования городов
- Поддержка взаимодействия между городскими элементами
- Координация действий внутри городов
- Обработка событий игроков в городах

## 📝 Новая структура событий

### Поддержка структурированных ID

```json
{
  "entity": {
    "id": "player-123",
    "type": "player",
    "name": "Вася"
  },
  "city": {
    "id": "city-456",
    "name": "Верхний Клан"
  },
  "quest_id": "quest-789"
}
```

## 🔄 Жизненный цикл

1. Получает события о городских сущностях через `world_events`
2. Обрабатывает запросы на управление
3. Организует координацию внутри города
4. Публикует результаты операций
5. Генерирует квесты для игроков

## 🧠 Состояние CityGovernor

- Отслеживает состояние городских структур
- Поддерживает координацию между элементами
- Не хранит долгосрочное состояние
- Работает с событиями в реальном времени

## 📡 Обработка событий

### Входящие:
- `city.governor.request` — запрос на управление
- `player.entry` — вход игрока в город
- `violation.detected` — нарушение правил города
- `quest.completion` — завершение квеста
- `npc.interaction` — взаимодействие с NPC

### Публикация событий:
- `city.population.updated` — обновление населения
- `quest.generated` — сгенерированный квест
- `npc.activated` — активация NPC

## 🌐 Интеграция

- **WorldGenerator**: создание городских структур
- **EntityManager**: управление сущностями
- **GameService**: взаимодействие с игроками
- **PlanManager**: планирование действий

## 🌐 Интеграция

- **WorldGenerator**: создание городских структур
- **EntityManager**: управление сущностями
- **EntityActor**: обработка действий игроков
- **BanOfWorld**: проверка нарушений
- **CultivationModule**: квесты для культивации
- **SemanticMemory**: контекст для повествования

## ✅ Преимущества

- Управление городскими структурами
- Координация внутри городов
- Интеграция с игровым процессом
- Гибкость в управлении
- **Новая структура событий**: поддержка `entity.id`

## 📊 Примеры событий

### Player Entry (входящее)
```json
{
  "entity": {
    "id": "player-123",
    "type": "player",
    "name": "Вася"
  },
  "city_id": "city-456"
}
```

### Quest Completion (входящее)
```json
{
  "entity": {
    "id": "player-123",
    "type": "player",
    "name": "Вася"
  },
  "quest_id": "quest-789",
  "city_id": "city-456"
}
```

### NPC Interaction (входящее)
```json
{
  "entity": {
    "id": "player-123",
    "type": "player"
  },
  "npc": {
    "id": "npc-456",
    "type": "npc",
    "name": "Городовой"
  },
  "interaction_type": "quest_giver"
}
```

### Violation Detected (входящее)
```json
{
  "entity": {
    "id": "player-123",
    "type": "player"
  },
  "violation_type": "noise_disturbance",
  "city_id": "city-456"
}
```

## 🛠️ Техническая реализация

- Сервис реализован в пакете `services/citygovernor`
- Использует `eventbus.EventBus` для подписки на события
- Подписывается на `eventbus.TopicWorldEvents`
- Обрабатывает события, связанные с городскими структурами

## 🔧 Конфигурация

- Переменные окружения: `KAFKA_BROKERS`
- По умолчанию: `localhost:9092`

## 📊 Мониторинг

- Количество обработанных городских событий
- Количество активных городов
- Время обработки запросов
- Эффективность координации
